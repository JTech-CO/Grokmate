소개

Grokmate는 xAI Grok 4.3 위에서 동작하는 디스코드 AI 봇입니다. 친근한 반말 대화부터 출처 기반 웹·X 검색, 코드 작성, 이미지·파일 분석, 그리고 권한을 가진 사람만 쓸 수 있는 자연어 서버 관리까지 한 봇 안에서 처리합니다.

모든 관리 액션은 봇이 아니라 요청한 사람의 권한으로만 실행되고, 성인 콘텐츠는 4중 게이트를 모두 통과해야만 열리도록 설계됐습니다.

대화 · AI

그록아, @멘션, DM으로 부르면 발화 단위로 모드를 판단해 답합니다. 기본은 반말 친근체.

  • 호출 — 웨이크워드·멘션·DM·전용 채널 — 메시지 맨 앞에 '그록아' 또는 'Hey Grok', 봇 멘션, DM, 또는 관리자가 지정한 전용 AI 채널(멘션 불필요) 중 하나면 응답하고 그 외 메시지는 무시합니다. 그록아 / Hey Grok / config ai-channel
  • 한국어·영어 전환 — 웨이크워드가 응답 언어를 정합니다. '그록아'는 한국어, 'Hey Grok'은 영어, 멘션·DM은 한국어가 기본. ko / en
  • 발화 단위 모드 판정 — 메시지마다 일반 / 검색 / 코딩을 규칙과 LLM 분류로 판단해 페르소나와 처리 경로를 바꿉니다. GENERAL · RESEARCH · CODING
  • 유저별 단기 대화 기억 — 채널×사용자 단위로 최근 메시지를 유지해 멀티턴 맥락을 잇습니다. 같은 채널이라도 유저 A의 SFW 코딩과 유저 B의 NSFW 대화·모드가 서로 섞이지 않습니다. 채널×유저당 20개 · 인메모리
  • 서버 맥락 인식 — 서버 이름·인원수·채널·역할·소유자 실값을 받아 추측 없이 답합니다. 요청자가 볼 수 있는 채널만 노출. 채널 60 · 역할 40
  • 작성 중 표시 — 답을 만드는 동안 'Grokmate가 답변을 작성 중입니다' 애니메이션을 띄우고, 완성되면 같은 메시지를 답변으로 교체합니다.

검색 · 연구 · 코딩

실시간 사실은 검색으로 근거를 대고, 코드는 길면 파일로 떨어뜨립니다.

  • 실시간 웹·X 검색 — 검색 모드에서 xAI 서버사이드 도구로 웹과 X를 검색해 답하고, 인용 출처를 본문과 분리해 각주로 보여줍니다. 출처 기본 3 · 최대 10
  • 긴 코드 파일 첨부 — 10줄 넘는 코드 블록은 언어에 맞는 확장자의 다운로드 파일로 빼고, 짧은 코드는 인라인으로 둡니다. 임계 10줄
  • /repo — 레포 검색 — 공개 GitHub 레포지토리를 검색해 별점순 상위 결과를 보여줍니다. 누구나 사용 가능. /repo query · 최대 8
  • /gh-user — 사용자 조회 — GitHub 사용자 프로필과 최근 업데이트된 레포 목록을 보여줍니다. /gh-user name · 레포 10
  • /gh-file — 파일·디렉터리 — 레포 파일을 인라인(길면 첨부)으로 보거나 디렉터리 개요를 훑습니다. /gh-file repo [path] · 400KB

외부 연동

링크를 안전하게 읽어 요약합니다. 명령어로도, 자연어로도.

  • /fetch — 페이지 읽고 요약 — 공개 웹 페이지를 안전하게 가져와 Grok 요약을 출처 링크와 함께 돌려줍니다. 읽기 전용이라 누구나. /fetch url [ask]
  • 자연어 링크 읽기 — 링크에 '읽어 / 요약' 신호를 주거나 링크가 사실상 메시지 전부일 때, 별도 명령 없이 페이지를 읽어 요약합니다. 첫 URL만 처리
  • 이미지·파일 분석 (SFW) — 이미지는 비전으로, 텍스트·코드 파일은 인라인으로 분석합니다. 항상 일반(SFW) 경로로만 동작. 이미지 10MB · 텍스트 256KB

서버 관리 · 모더레이션

관리 액션은 요청한 사람의 권한으로만 실행됩니다. 위험한 작업은 슬래시 전용·확인 버튼.

  • 자연어 서버 관리 (권한 필요) — 슬로우모드 설정, 채널 생성, 역할 부여·해제는 권한이 되는 사람이 자연어로 지시하면 실행됩니다. 슬로우모드 0–21600초
  • 고위험은 슬래시 전용 + 확인 — 밴·킥·타임아웃·청소는 채팅으로 실행하지 않고 슬래시 명령으로 유도하며, 확인 버튼 뒤에서 실행합니다. /ban /kick /timeout /purge
  • 자동 모더레이션 (옵트인) — 관리자가 켜면 스팸·욕설·외부 링크 규칙에 걸린 메시지를 삭제·경고 후 기록합니다. 기본 비활성 · 허용 목록 외 링크 차단
  • 서버 설정 · 경고 — 전용 AI 채널·존댓말·자동 모더·역할·모더 로그 채널을 한 명령으로 관리하고, 경고는 누적 시 자동 타임아웃합니다. /config · /warn · /pardon
  • XP 레벨링 + 리더보드 — 채팅으로 XP를 쌓아 레벨업하면 안내가 뜨고, 슬래시로 레벨·순위를 확인합니다. /level · /leaderboard
  • 24/7 운영 · 점검 모드 — 점검·업데이트 시 켜두면 모든 요청에 '점검 중' 안내가 나가고, 단일 인스턴스 잠금·우아한 종료로 무중단성을 지킵니다. /maintenance · /health
  • 서버 합류 온보딩 — 봇이 서버에 들어오면 기본 설정을 만들고 호출 방법을 설명하는 한국어 환영 메시지를 보냅니다.

안전 · 신뢰

불변식을 코드로 강제합니다 — 인가는 요청자 권한 단일 게이트, 미성년·NSFW 누출은 0. 약속이 아니라 테스트로 고정된 규칙입니다.

  • 단일 인가 게이트 (INV-1) — 모든 관리 액션은 슬래시든 자연어든 동일한 authorize() 게이트를 거치고, 권한은 봇이 아니라 요청자에서 도출합니다. 미등록 액션은 자동 거부.
  • 전수 감사 로그 — 허용·거부를 가리지 않고 모든 관리 결정을 행위자·액션·대상·결과와 함께 남깁니다.
  • NSFW 4중 게이트 (INV-3) — 성적 의도·연령 제한 채널·서버 활성·연령 인증을 모두 통과해야만 성인 모드가 열립니다. 하나라도 실패하면 일반 모드. 따로 켜는 토글은 없고, 막힌 게이트가 있으면 무엇이 필요한지 안내합니다.
  • 하드차단 필터 (INV-2) — 미성년 성적·비동의·실존 인물 딥페이크·수간·성+고어는 게이트를 다 통과해도 무조건 거부합니다. 입력·출력 양쪽 검사.
  • 연령 인증 — /verify-age로 19세 이상임을 1회 자가 인증하면 Discord 연령 게이트와 결합해 본인에게만 비공개로 기록됩니다.
  • 인젝션·스니핑 방어 — 시스템 프롬프트나 도구 목록을 빼내려는 시도, '이전 지시 무시' 류는 안전 경로로 우회시키고 기능 요약을 돌려줍니다.
  • Fetch 가드레일 — 가져오는 모든 URL은 사설/예약 IP·DNS 리바인딩 차단, robots.txt 확인, 사용자별 레이트리밋, 크기·시간·리다이렉트 상한을 거칩니다. 2MiB · 10s · 5/30s

정직한 한계

모르는 건 모른다고 하는 게 원칙입니다. 과장하지 않으려고 한계도 함께 적습니다.

  • 실시간 정보는 검색 모드에서만 가져옵니다. 일반 대화는 웹·X 검색을 돌리지 않아 최신 사실은 모를 수 있습니다.
  • 이미지·파일 분석은 SFW 전용이며, PDF와 바이너리는 v1에서 지원하지 않습니다. 이미지는 정규식으로 검사할 수 없어 모델 안전성과 출력 필터에 의존합니다.
  • 웹 페이지 읽기는 본문을 휴리스틱으로 추출하므로 JS 렌더링·로그인이 필요한 페이지는 읽지 못하고, 그럴 땐 지어내지 않고 못 읽었다고 솔직히 말합니다.
  • 대화 기억과 일부 레이트리밋·쿨다운 상태는 인메모리라 재시작하면 초기화되고 샤드 간 공유되지 않습니다.
  • 안전 필터(하드차단·스니핑 탐지)는 정규식·키워드 기반이라 새로운 표현이나 미지원 언어를 놓칠 수 있습니다. 연령 인증도 자가 인증과 Discord 연령 게이트일 뿐 신분증 검증은 아닙니다.
