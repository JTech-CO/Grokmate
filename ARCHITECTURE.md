# 아키텍처 개요 (Architecture)

> 개념 수준 개요입니다. **봇 구동 소스는 비공개**이며, 정확한 스키마·내부 인가 매트릭스·필터 규칙 등은 포함하지 않습니다.

Grokmate는 "기능을 더하기 전에 경계와 불변식을 먼저 고정한다"는 원칙으로 설계됐습니다. 인가·안전·정직성을 사람이 지키는 규칙이 아니라 **코드와 테스트로 강제**합니다.

## 레이어 경계

모듈은 단방향 의존으로 배치되고, 위반은 lint(`eslint-plugin-boundaries`)로 차단됩니다.

```
config  →  grok · github · web · ocr · billing · db · auth  →  discord · moderation · nsfw · tools  →  core
```

- **config** — 환경/시크릿 로딩의 유일한 지점.
- **grok** — xAI Grok API 호출의 유일한 진입점.
- **github / web** — 외부 데이터(공개 레포 / 임의 URL) 접근 전용 모듈.
- **ocr** — 이미지 텍스트 추출(로컬 WASM). **billing** — 요금제·토큰 쿼터.
- **db** — 영속·캐시(리포지토리 패턴).
- **auth** — 인가 단일 통제점(순수 판정). nsfw/moderation/tools 를 import 하지 않음.
- **discord / moderation / nsfw / tools** — 어댑터·기능 레이어.
- **core** — 오케스트레이터(인입 → 분류 → 라우팅 → 실행 → 출력).

## 처리 흐름 (요약)

1. **인입 게이트** — 봇·웹훅 무시, 호출어·멘션·DM·전용 AI 채널만 처리. 점검 모드 시 안내로 단락.
2. **하드 차단(입력)** — 금지 카테고리는 분류·저장·생성 전에 즉시 거절.
3. **발화 단위 모드 분류** — 규칙 1차 + 모호 시 LLM 2차(보수적). 성적·관리 오탐 억제 — 짧은 관리 동사(밴/킥/청소)는 **낱말 경계**일 때만 인식해 '밴드'(band)·'청소년'을 밴·청소로 오해하지 않음.
4. **라우팅** — 일반 / 검색 / 코딩 / 서버 관리 / 성인 / 첨부 / URL 읽기.
5. **생성 + 출력 안전 검사** — 성인·멀티모달은 비스트리밍 생성 후 **전송 전** 출력 하드차단(누출 0).

대화 맥락은 **채널×사용자** 단위 단기 인메모리 윈도우로 분리됩니다(유저 간·모드 간 비혼합).

## 데이터 (개념 수준)

운영용 SQLite에 저장: 서버 설정, 연령 인증 상태(불리언 + 시각, 신원 정보 아님), 레벨/XP, 관리 감사 로그, 일일 토큰 사용량·요금제 상태, 경고·투표·반응 역할. 대화 맥락은 비영속(인메모리). 점검 상태는 별도 플래그 파일.

## 보안 모델

7개 불변식(INVARIANTS)을 테스트로 고정합니다.

| ID | 내용 |
|---|---|
| **INV-1 권한 상승 0** | 관리 액션은 봇이 아니라 *요청자* 권한으로, 슬래시·자연어 모두 **단일 `authorize()` 게이트**를 거침. 모델이 함수를 호출했다는 사실은 '의도'일 뿐 '권한'이 아니며, 매트릭스 미등록 액션은 자동 거부(default-deny). |
| **INV-2 미성년 성적 콘텐츠 0** | 모든 게이트·모드·우회 시도에서 예외 없이 차단(최우선). |
| **INV-3 NSFW 누출 0** | 채널(연령제한)·서버(활성)·사용자(연령 인증)·의도(발화 단위) **4중 게이트** 통과 시에만 성인 모드. |
| **INV-4 정직성** | 도구 거부를 성공으로 위장 0, 출처 없는 단정 억제, 동문서답 억제. |
| **INV-5 콘텐츠 안전** | 성적 콘텐츠에 폭력·고어 결합 차단. |
| **INV-6 시크릿 보호** | API 키·토큰이 코드·로그에 노출 0. |
| **INV-7 역할 위계** | 자신보다 상위 역할 대상 액션 0. |

추가 가드:

- **전수 감사 로그** — 허용·거부를 가리지 않고 모든 관리 결정을 기록.
- **하드차단 필터** — 입력·출력 양쪽 검사, 정규화(NFKC·제로폭 제거)로 난독화 우회 차단.
- **인젝션/스니핑 방어** — 시스템 프롬프트·도구 정의 비노출, 누출 의심 출력 재검증.
- **역할 부여 재검증** — 자동 역할·반응 역할은 설정 시점뿐 아니라 **적용 시점**에도 위계·위험권한을 재검증(TOCTOU 드리프트 차단).
- **봇 멘션 대여 차단** — 봇에게 타 유저·`@everyone` 멘션을 시키는 요청은 스탭 이상만. 봇의 모든 출력에서 `@everyone`/`@here`/역할 핑은 전역 차단.
- **Fetch 가드레일** — 사설/예약 IP·DNS 리바인딩(연결 시점 재검증)·robots.txt·사용자별 레이트리밋·크기/시간/리다이렉트 상한.

## 운영 견고성 (24/7)

- **단일 인스턴스 잠금** — 중복 프로세스가 한 토큰에 붙어 응답이 겹치는 것을 방지.
- **우아한 종료** — 종료 신호에 게이트웨이 정상 해제 + DB 체크포인트/닫기(배포 재시작 안전).
- **업스트림 타임아웃** — 응답 없는 AI·OCR·다운로드가 자원을 영구 점유하지 못하도록 시간 상한·자동 회수.
- **점검 모드** — 점검·업데이트 시 켜두면 봇은 켜진 채로 모든 요청에 '점검 중' 안내(재시작 넘어 유지). 운영자 관측 명령(비용·상태·감사)으로 무인 운영 가시성 확보.

## 품질 보증

- JavaScript + JSDoc, `tsc --checkJs --strict` 타입 그린.
- 모듈 경계 lint 강제(우회 직접 실행 차단).
- Vitest 584개 + 결정적·무비용 불변식 게이트(인가 parity·NSFW 안전): 하드차단 차단율 100%·4중 게이트 누출 0·슬래시↔자연어 인가 parity 100%.
- 게이트: lint + 타입체크 + 테스트 + 불변식 게이트를 통과해야 머지. **GitHub Actions CI**가 매 푸시·PR에서 자동 검증.

## 기술 스택

Node.js 20+ (ESM) · discord.js v14 · xAI Grok 4.3(`@ai-sdk/xai`, Vercel AI SDK) · SQLite + Drizzle ORM · tesseract.js(OCR) · undici · ESLint(+ boundaries) · Prettier · Vitest · GitHub Actions.
