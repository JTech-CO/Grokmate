# ADR 0003 — 안전 · NSFW 게이팅 정책

- **상태**: 채택 (Accepted)
- **날짜**: 2026-06-11
- **불변식**: INV-2(미성년 0), INV-3(NSFW 누출 0), INV-5(콘텐츠 안전)
- **관련 미결 질문**: Q1(xAI AUP의 NSFW 허용 범위) — 본 ADR로 해소
- **상세 설계**: 기술백서 §7, 디자인백서 §6

## 맥락

요구사항 2는 "Grok의 성인 모드 등 제한이 없는 점을 살려 NSFW 수위 대화"를, 요구사항 6은 "일상 질문에 19금 답변 금지"를, 요구사항 7은 "NSFW는 오직 성인물"을 요구한다. 이를 실현 가능하면서 안전·합법적으로 설계해야 한다.

### Q1 조사 결과 (xAI Acceptable Use Policy, 2025-01-02 발효, 개발자·기업 포함)

AUP가 **명시적으로 금지**하는 것: 인물의 포르노적 묘사(실존인물 likeness), **아동 성적 대상화·착취**(CSAM, NCMEC 신고), IP·프라이버시 침해, 사기·명예훼손·스토킹, 대량살상무기, **안전장치 우회**.

AUP가 **금지하지 않는 것**: 텍스트 기반 합의 성인 콘텐츠, 성인 금기 판타지 픽션(미성년·실존인물 포르노·불법 요소를 포함하지 않는 한).

중요 단서: 출력은 **여전히 모더레이션**되며, API로 모더레이션을 우회하는 공식 방법은 없다. 즉 "제한 없음"이 아니라 **"텍스트 성인물에 관대하나 무제한 아님 + 모더레이션 존재"** 가 정확한 사실이다.

## 결정

1. **텍스트 성인 모드는 채택하되 무제한 모델로 설계하지 않는다.** 모델이 허용하는 범위 안에서 동작하고, 거부 시 우아한 폴백.
2. **안전장치 우회 금지(AUP 준수)**: 시스템 프롬프트를 탈옥으로 설계하지 않는다. 이는 AUP 위반이자 INV에 반한다.
3. **하드 차단(불변·예외 없음)** — AUP 금지 카테고리와 정렬:
   - **미성년 성적 콘텐츠 0**(INV-2, 최우선). 연령 모호 정황도 차단.
   - 실존인물의 포르노적 묘사·딥페이크 차단.
   - 비동의·착취·불법 콘텐츠 차단.
   - 성적 맥락과 폭력/고어 결합 차단(INV-5). NSFW는 *성인물에 한함*.
4. **4중 게이트**(통과해야만 성인 모드): 채널(Discord age-restricted) + 서버(관리자 활성) + 사용자(연령 인증) + 의도(발화 단위 성적).
5. **발화 단위 모드 판정**(INV-3): 채널이 NSFW여도 비성적 발화는 일반 모드. "오늘 뭐 먹지"에 19금이 새지 않음.
6. **자체 필터 병행**: 모델 거부에만 의존하지 않고 입력·프롬프트·출력 3중 필터로 하드 차단을 강제(우회/인젝션 대비).
7. **실측 의무**: 정책 허용 ≠ 모델이 실제 생성을 보장. M1/M7에서 실제 API 거동을 평가셋으로 검증.

## 대안

- **무제한 성인 모델로 전제**: AUP·모더레이션 현실과 불일치, 탈옥은 ToS 위반 → 기각.
- **NSFW 전면 배제**: 요구사항 2 미충족 → 기각.
- **채널 단위로만 모드 고정**: 같은 채널의 일상 발화에 19금 누출(INV-3 위반) → 발화 단위 판정으로 대체.

## 결과

- M7에서 4중 게이트·연령 인증·하드 차단 다층 필터를 구현. 진입조건 = M3(모드분류)+M4(권한) 게이트.
- M7 게이트(절대): 하드 차단 차단율 **100%**(미성년·우회 포함), 비게이팅 누출 **0**. 99%는 실패.
- 연령 인증 방식(단순 확인 vs 강한 검증)은 Q3에서 결정.
- 운영 시 Discord ToS(연령 제한 채널 한정)·국내 청소년보호법 관점 점검.
- 요구사항 2의 "제한 없음" 전제는 *텍스트 성인물에 관대하나 모더레이션 존재*로 정정해 사용자와 합의.
