# ADR 0001 — 기술 스택 선택

- **상태**: 채택 (Accepted)
- **날짜**: 2026-06-11
- **관련 미결 질문**: Q2(discord.js 버전), Q4(툴체인)

## 맥락

Grok 4.3 기반 디스코드 봇을 1인 개발자가 JS로 구축한다. 서버 관리·범용 AI·게이팅된 성인 모드를 한 봇에 통합하며, 보안·안전 불변식을 코드 레벨에서 강제해야 한다(CLAUDE.md). 런타임·언어·라이브러리·툴체인을 확정한다.

## 결정

- **런타임**: Node.js LTS(≥20), ESM(`"type": "module"`).
- **언어**: JavaScript + JSDoc, 타입 검증은 `tsc --noEmit --checkJs`. (주력 언어 JS를 유지하면서 타입 안전 확보.)
- **디스코드 SDK**: **discord.js v14(14.26.x)**. 음성 필요 시 `@discordjs/voice`.
- **LLM**: xAI **Grok 4.3**. JS 통합은 Vercel AI SDK(`@ai-sdk/xai`) 1순위, 대안으로 OpenAI SDK + base_url. (M1에서 도입.)
- **DB/캐시**: PostgreSQL + Redis. (초기 SQLite 가능 — Q5에서 결정.)
- **패키지 매니저**: **pnpm**.
- **테스트**: **vitest**.
- **린트/포맷**: ESLint(flat config) + **eslint-plugin-boundaries**(모듈 경계 강제) + Prettier.

## 근거

- discord.js **v15는 프리릴리스**라 프로덕션 권장 외. 안정 라인 v14가 정답(Q2 해소).
- JS + JSDoc + checkJs는 TS 전환 비용 없이 타입 안전을 얻고, 주력 언어를 유지한다(Q4).
- eslint-plugin-boundaries로 CLAUDE.md §2.1 의존 방향과 §2.2 인가 통제점을 lint로 강제 → "사람이 지키는 규칙"이 아니라 "도구가 막는 규칙"으로 격상.
- pnpm·vitest는 빠르고 1인 다(多)프로젝트 운영에 효율적.

## 대안

- **TypeScript 전면 도입**: 타입 안전 최상이나 주력 언어 JS 선호와 빌드 단계 추가 비용 → JSDoc+checkJs로 절충.
- **discord.js v15(프리릴리스)**: 최신 API지만 프로덕션 불안정 → 보류, 안정화 후 마이그레이션(별도 ADR).
- **npm/jest**: 무난하나 속도·DX에서 pnpm/vitest 대비 열위. npm은 폴백으로 허용.

## 결과

- `pnpm install` → `pnpm run check`(lint+typecheck+test)가 M0 게이트.
- 버전 핀은 설치 시 해소되며, 최신화는 `pnpm up --latest` + 회귀 테스트로.
- eslint-plugin-boundaries 설정은 설치된 플러그인 버전에 맞춰 1회 점검(M0).
- v15 안정화·AI SDK 메이저 변동은 재검토 트리거.
