# 명령어 (Commands)

## 호출 방법

- **웨이크워드** — 메시지 맨 앞에 `그록아`(한국어) 또는 `Hey Grok`(영어)
- **멘션** — `@Grokmate ...`
- **DM** — 봇에게 다이렉트 메시지
- **전용 AI 채널** — 관리자가 `/config ai-channel add` 로 지정한 채널에서는 멘션·웨이크워드 없이 그냥 말하면 응답
- **자연어 링크 읽기** — 링크에 "읽어 / 요약"을 덧붙이면 페이지를 가져와 요약
- **자연어 서버 관리(권한 필요)** — "슬로우모드 10초로", "역할 부여" 등

응답 언어는 웨이크워드를 따릅니다(`그록아`→한국어, `Hey Grok`→영어, 멘션·DM→한국어).

## 누구나 쓰는 슬래시 명령

| 명령 | 설명 |
|---|---|
| `/help` | 명령어·기능 안내(본인만 표시) |
| `/reset` | 나와 나눈 대화 맥락(기억) 초기화 |
| `/ping` | 봇 응답·지연 확인 |
| `/level [user]` | 레벨·XP 확인(대상 생략 시 본인) |
| `/leaderboard` | 서버 레벨 순위(상위 10) |
| `/poll <질문> <선택지…>` | 투표/설문(버튼 집계, 다중·비밀·자동마감 옵션) |
| `/verify-age` | 만 19세 이상 자가 인증(성인 채널 이용용, 본인만 표시) |
| `/usage` | 내 토큰 사용량·요금제 확인 및 업그레이드(Basic/Pro/Max) |
| `/repo <검색어>` | GitHub 공개 레포 검색(별점순 상위) |
| `/gh-user <유저명>` | GitHub 사용자 프로필·레포 목록 |
| `/gh-file <owner/repo> [경로]` | 레포 파일 내용·디렉터리 개요 |
| `/fetch <url> [ask]` | 웹 페이지를 읽어 요약(출처 포함) |
| `/translate <이미지> [대상언어]` | 이미지 속 글자를 OCR 추출 후 번역 |
| `/voice <name\|limit\|lock\|unlock>` | 내가 만든 임시 음성채널 제어(방장) |

## 관리자·스탭 전용

> 권한이 없으면 명령 목록에 표시되지 않습니다.

**모더레이션 (ModerateMembers 등)**

| 명령 | 설명 |
|---|---|
| `/timeout <user> <분> [사유]` | 타임아웃 |
| `/untimeout <user> [사유]` | 타임아웃 조기 해제 |
| `/kick <user> [사유]` | 킥 |
| `/ban <user> [사유]` | 밴 (확인 버튼) |
| `/unban <user_id> [사유]` | 차단 해제(Discord ID로 지정) |
| `/purge <개수>` | 메시지 청소 (확인 버튼) |
| `/warn <user> <사유>` | 경고 부여(누적 시 자동 타임아웃) |
| `/warnings <user>` | 경고 이력 조회 |
| `/pardon <user> [번호\|count]` | 경고 사면 — 특정 번호·최근 N건(개수 줄이기)·전체 |

**채널 관리 (ManageChannels)**

| 명령 | 설명 |
|---|---|
| `/slowmode <초>` | 채널 슬로우모드(0–21600초) |
| `/create-channel <이름> [종류]` | 채널 생성(텍스트/음성) |
| `/delete-channel <채널>` | 채널 삭제 (확인 버튼) |
| `/lock` · `/unlock` | 현재 채널 잠금/해제 |

**서버·역할 설정 (ManageGuild / ManageRoles)**

| 명령 | 설명 |
|---|---|
| `/config <status\|reset\|honorific\|manage-via-nl\|automod\|ai-channel\|log-channel\|staff\|admin>` | 서버 설정 — 전용 AI 채널·존댓말·자연어 관리·자동 모더레이션·모더 로그 채널·스탭/관리 역할·전체 초기화 |
| `/welcome <set\|goodbye\|off\|status\|test>` | 가입/퇴장 환영·작별 메시지 |
| `/autorole <set\|off\|status>` | 가입 시 자동 역할 |
| `/reactionrole <add\|remove\|list>` | 메시지 반응 → 역할 자동 부여 |
| `/voice-hub <set\|off\|status>` | 임시 음성채널 허브(Join-to-Create) |
| `/nsfw-policy <on/off>` | 서버 성인 모드 활성/비활성 (확인 버튼) |

밴·킥·타임아웃·청소는 채팅(자연어)으로 실행하지 않고 **슬래시 명령 + 확인 버튼**으로만 실행됩니다. 슬로우모드·역할 부여/회수·채널 생성 등은 권한자가 자연어로도 지시할 수 있습니다.

## 봇 운영자 전용

> 봇을 직접 운영하는 사람(설정된 운영자 ID)만 사용할 수 있으며, 서버 관리자 권한만으로는 실행되지 않습니다.

| 명령 | 설명 |
|---|---|
| `/maintenance <on/off> [message]` | 점검 모드 — 켜면 점검이 끝날 때까지 모든 대화·명령에 "점검 중입니다" 안내(재시작해도 유지) |
| `/cost` | 토큰 사용량 집계(오늘·최근 7일) |
| `/health` | 봇 상태(동시성·OCR 대기·업타임·메모리·점검 모드) |
| `/audit` | 최근 권한(관리 액션) 감사 로그 |
| `/grant-tier`·`/grant-list` | 유료 티어 수동 부여·활성 구독 조회 |

모든 관리 액션은 **요청자 권한으로 단일 인가 게이트를 거치고 전수 감사 로그**에 기록됩니다. `/config log-channel` 지정 시 자동 모더레이션 삭제·경고·사면이 로그 채널에도 임베드로 기록됩니다. 자동 역할·반응 역할은 관리 권한이 있는 역할을 지급 대상으로 설정할 수 없습니다(권한 상승 차단). 봇에게 다른 유저·@everyone 멘션을 시키는 요청은 스탭 이상만 가능하며, 봇 출력에서 @everyone/@here/역할 핑은 항상 차단됩니다(봇 권한 대여 방지).
